1. 项目概览

本项目旨在构建一套高可靠、去中心化的光伏能源监控系统。系统通过 RS485 接口实时采集光伏控制器的核心数据（电压、功率、累计发电量），利用国密 SM2 算法进行数字签名，并通过安全加密隧道将数据存证至由全球多点部署的 FISCO-BCOS 联盟链上，实现数据的不可篡改与永久追溯。

3. 项目实施里程碑

第一阶段：驱动开发与本地采集

  • 基于 pymodbus 开发了 SolarMonitor 类，实现了对 EPEVER 控制器各项寄存器的精准读取。
  • 重点攻克了 累计发电量 (Total Generated Energy) 的读取，这是 RWA (现实世界资产) 最核心的价值凭证。

第二阶段：全球节点部署

我们构建了一个跨地域的高可用联盟链网络： 1. Gwen (Master): 部署于腾讯云，承载核心 write 请求。 2. Miles: 部署于腾讯云（异地），作为共识验证节点。 3. Peter: 部署于 DigitalOcean (新加坡)，实现数据的跨境容灾与验证。 所有节点通过 PBFT 共识算法保持数据强一致性。

第三阶段：合约开发与安全上链

  • 开发 Solidity 智能合约 SolarDataStore.sol，定义了不可篡改的数据结构。
  • 实现了 ChainClient Python SDK，并通过 双通道 SSH 隧道 (RPC + Relay) 解决了云服务器防火墙与本地内网穿透的安全问题。

4. 系统优化与重构 (最新成果)

基于初期 8 小时挂机测试的数据反馈，我们进行了深度优化：

4.1 稳定性提升 (P0)

  • Relay v2: 部署了新版中间件，增加了“控制台唤醒机制”，彻底解决了服务端空响应导致的丢包问题。
  • 自动重连: 客户端增加了 SSH 隧道状态监测，一旦断连会自动重试，确保 7x24 小时在线。
  • 时间校准: 采样逻辑从“相对休眠”改为“绝对时间调度”，消除了长期运行的采样时间漂移。

4.2 安全与维护 (P1)

  • 配置中心化: 所有密钥、路径、IP 地址统一提取至 config.py，并支持环境变量注入，杜绝了硬编码的安全隐患。
  • 目录重构: 建立了 data/ (数据), charts/ (图表), utils/ (工具) 的清晰目录结构，便于长期运维。

4.3 用户体验 (P2)

  • 数据可视化: 优化了 Matplotlib 绘图引擎，支持深色模式，自动生成涵盖 PV/电池/负载曲线的高清图表。
  • 自动关灯: 增加了脚本退出时的安全钩子，确保测试结束后自动切断负载电源。

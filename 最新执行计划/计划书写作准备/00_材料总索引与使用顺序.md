## 二、旧项目中可用的主要材料

### 1. 旧校级申请书

原始文件：

- `../../项目总结报告内容/大学生创新创业训练计划创业训练项目申请书(1).pdf`

文本提取版：

- `旧校级大创申请书_文本提取.txt`

可用内容：

- 项目负责人、团队成员、导师、学院、项目类型等基础信息。
- 负责人曾参与科研 / 竞赛情况。
- 指导教师相关科研课题情况。
- 旧项目的团队分工框架。
- 部分经费预算科目。

不能直接照搬的内容：

- RWA 资产交易平台叙事。
- 两链一桥跨境融资。
- 代币、认购、收益、分红、DEX、稳定币等内容。

### 2. 项目技术报告

原始文件：

- `../../项目总结报告内容/SolarGuard Project Report.pdf`

文本提取版：

- `SolarGuard项目报告_文本提取.txt`

可用内容：

- 光伏控制器 RS485 / Modbus 数据采集。
- 国密 SM2 / SM3 签名。
- FISCO-BCOS 联盟链多节点存证。
- `SolarDataStore.sol` 合约。
- `chain_client.py`、`auto_monitor.py`、`solar_monitor.py` 等代码模块。
- 2026-02-19 8 小时连续测试、96 个采样点。
- Telegram 告警、图表生成、云端部署等工程成果。

使用方式：

- 写进“项目当前进展”“生产或运营”“技术水平”“预期成果基础”。
- 不要照搬“RWA 资产上链系统”标题，应改成“历史原型系统 / 可信数据采集与存证原型”。

### 3. 历史 RWA 版本报告

原始文件：

- `../../项目总结报告内容/光伏RWA项目.pdf`

文本提取版：

- `光伏RWA项目_文本提取_历史风险版.txt`

用途：

- 只用于识别旧风险表达和历史技术背景。
- 不建议作为新申请书正面材料直接引用。

可保留的表达：

- 曾开展绿色能源资产数字化映射技术探索。
- 已完成 IoT + 国密 + 联盟链的原型验证。

必须降级的表达：

- ERC-20、代币、DEX、稳定币、Mint、资产化铸造、交易与收益。

## 四、代码材料的使用方式

可直接引用的代码基础：

- `../../solar_monitor.py`：真实光伏数据采集、SM2 签名、模拟模式。
- `../../auto_monitor.py`：定时采样、CSV 落盘、图表生成、Telegram 推送、链上同步。
- `../../chain_client.py`：FISCO-BCOS 客户端、链上写入和查询。
- `../../contracts/SolarDataStore.sol`：纯数据存证合约。
- `../../api/app.py`：已有 API 服务雏形。
- `../../frontend/src/app/page.tsx`：前端可视化看板雏形。

写作时的准确口径：

- “已完成可信采集、签名、存证、监测和展示原型。”
- “MRV 核算引擎和第三方核查接口是下一阶段重点建设任务。”

不要写成：

- “已经完成完整 CCER 核查系统。”
- “已经被核查机构正式采信。”
- “已经实现真实商业收费闭环。”

## 五、正式写作前的最低准备

开始写 `05_国省级大创项目申请书_创业训练项目_正文稿.md` 前，应先确认：

- 项目名称最终版本。
- 项目周期：2026.05-2027.05。
- 团队成员和导师信息是否沿用旧申请书。
- 经费预算是否仍按 6000 元，或调整为国省级更合理版本。
- 是否保留“绿链未来”作为中文项目名，`SolarMRV` 作为技术品牌名。
- 是否要在材料中提及 GitHub 仓库名仍含 RWA，并用一句话解释为历史链接保留。

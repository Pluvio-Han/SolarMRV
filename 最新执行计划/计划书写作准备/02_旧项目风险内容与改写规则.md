# 旧项目风险内容与改写规则

> 用途：防止新申请书把旧 RWA 版本里的高风险表达带进去。  
> 原则：旧内容不是全部否定，而是把金融化、交易化、收益化部分降级为历史技术探索，把可信数据和工程验证部分保留下来。

---

## 一、必须删除或避免作为主线的表达

以下词语不建议出现在项目名称、项目简介、项目特色、商业模式和预期成果的主线中：

- RWA 资产交易平台
- 现实世界资产代币化平台
- 代币 / 通证 / Token
- Mint / 铸币 / 自动铸造
- DEX / 去中心化交易所
- 稳定币 / mUSD
- SLG 代币
- 跨境融资
- 香港金融链
- 两链一桥作为金融流转架构
- 双 SPV
- 收益权拆分
- 面向大学生小额认购
- 光伏云认养收益
- 分红
- 公开交易
- 兑换回稳定币
- 全球 Web3 绿色投资基金募资

原因：

- 国省级大创材料属于学校和省教育系统正式申报文件。
- 通知要求在已立项校级项目基础上升级，评审首先看合规性和可执行性。
- 当前大陆监管环境下，上述表达容易被理解为代币发行、公众募资、类证券化流转或金融中介。

---

## 二、可以保留但必须降级的表达

| 旧表达 | 新写法 |
|---|---|
| 光伏 RWA 项目 | 早期围绕绿色能源资产数字化映射开展的技术探索 |
| 代币化 | 环境权益数据的数字化记录与可信映射 |
| 铸币 / Mint | 生成不可转让的减排量电子记录 |
| 交易 / DEX | 数据查询、核查接口、审计留痕 |
| 稳定币结算 | 不再作为当前产品功能表述 |
| 绿色资产流转 | 绿色资产核验、MRV 报表输出、辅助披露 |
| 两链一桥 | 历史探索，不作为本次申报主架构 |
| 跨境融资 | 删除，改为 CBAM 绿电使用证明或出口企业合规辅助 |
| 面向学生认购 | 删除，改为校园 / 园区 PoC 示范 |
| 收益兑现 | 删除，改为减排贡献可视化和教育实效 |

## 三、旧申请书各部分的风险判断

### 1. 项目名称

旧名称：

> 绿链未来：基于“两链一桥”的分布式光伏 RWA 资产数字化管理平台

问题：

- RWA 和“两链一桥”直接把项目带向跨境金融和代币化。
- “资产数字化管理平台”容易被理解为金融化资产平台。

建议新名称：

> 绿链未来：基于国密与联盟链的分布式光伏 MRV 可信数据底座

或：

> SolarMRV：面向分布式光伏的减排量可信核算与碳资产数据底座

### 2. 项目简介

旧简介高风险点：

- “打通境内实体资产与国际合规资本通道”
- “两链一桥跨境合规架构”
- “双 SPV 风险隔离模型”
- “收益权数字化碎片拆解”
- “参与门槛降至 50 元”
- “透明收益”

新简介应强调：

- 分布式光伏 MRV 数据可信问题。
- 设备直采、SM2 签名、联盟链存证、减排量核算。
- 服务第三方核查、园区碳管理、CCER 申报咨询、ESG / CBAM 辅助证明。
- 不发币、不交易、不募资。

### 3. 项目概况

旧概况高风险点：

- “资产证券化双重共振”
- “香港 RWA 注册登记制度”
- “绿色能源普惠交易平台”
- “可拆分、可交易、可验真的标准化数字权益”

新概况应改为：

- 双碳、CCER 重启、全国碳市场扩容、分布式光伏规范化。
- MRV 核查成本高、底层数据可信度不足。
- 项目提供可信数据采集、存证和减排量核算工具。

### 4. 管理模式

旧内容可部分保留：

- CEO / CTO / CFO / CMO / CDO 分工框架。

需要改写：

- CFO 不再负责 RWA 资产定价和碳积分经济模型。
- CMO 不再负责校园社群裂变和认购。
- 改为政策研究、试点对接、核查机构接口调研、材料与展示。

### 5. 生产或运营

旧内容可继承：

- 硬件级数据采集。
- 数据签名与加密封装。
- 联盟链存证。
- 校园沙盒 / 校园试点想法。

必须删除：

- 香港金融链。
- ERC-3525 / ERC-20。
- 公链价值流通层。
- 自动分红。
- 学生资金认购。

新运营应写：

- 校园或园区分布式光伏 PoC。
- 设备采集网关部署。
- MRV 报表生成。
- 第三方核查接口演示。
- 数据审计日志。

### 6. 投融资方案

旧内容基本不宜照搬。

旧高风险点：

- 天使校友融资。
- 供应链金融与资产置换。
- 合规 RWA 发行。
- 香港发行标准化权益凭证。

新写法：

- 大创经费、学校配套资源、试点硬件采购、云服务器、软著专利费用。
- 后续如有外部合作，表述为“项目服务费、接口订阅、园区部署服务”，不是融资。
- 本申请书中更适合写“资源需求与资金使用计划”，而不是“融资计划”。

### 7. 效益预测

旧内容需重写。

旧风险点：

- 平台撮合交易额。
- 收益分成。
- 管理资产规模。

新写法：

- 经济效益：MRV 数据服务、接口服务、园区部署的保守收入测算。
- 社会效益：降低核查成本、提高减排数据可信度、支撑双碳教育和校园示范。
- 教育效益：跨学科训练、工程实践、政策合规意识。

### 8. 风险预测

旧内容可继承“合规意识”，但要改掉高风险方案。

旧风险写法：

- 内地和香港实体隔离。
- 跨境融资在沙盒中探索。

新风险写法：

- 政策口径变化：持续跟踪 CCER、绿证、数据要素政策。
- 技术稳定性：断点续传、数据校验、测试覆盖。
- 试点落地：先做校园 / 园区 PoC。
- 数据安全：权限控制、脱敏、API Key。
- 合规红线：不发币、不交易、不募资、不承诺收益。

## 四、新申请书统一安全口径

建议全文反复保持以下边界：

> 本项目不发行任何金融化代币，不撮合任何交易，不开展公众募资，不向用户承诺收益。项目链上记录仅作为分布式光伏发电数据和减排量核算过程的可信留痕，不具备转让、流通或金融投资属性。

如需解释历史代码：

> 项目早期曾围绕绿色能源资产数字化映射开展技术探索，相关历史代码仅作为原型研究留存。根据最新监管环境和本次国省级大创申报要求，项目已主动完成合规调整，正式转向分布式光伏 MRV 可信数据底座。

## 五、写作时的自检清单

写完任一章节后，用下面清单检查：

- [ ] 是否把项目主线写成 MRV 可信数据底座？
- [ ] 是否避免把 RWA、代币、DEX 写成当前主营业务？
- [ ] 是否写清楚不发币、不交易、不募资？
- [ ] 是否能对应现有真实代码和实验？
- [ ] 是否区分“已完成成果”和“下一阶段计划”？
- [ ] 是否避免夸大为已经完成真实 CCER 签发或机构采信？
- [ ] 是否把资金需求写成大创经费和试点资源，而不是金融融资？

# 新旧项目映射表

> 用途：保证国省级申请书既能体现“从校级项目升级而来”的延续性，又能完成 MRV 合规转向。  
> 核心策略：不是否定旧项目，而是把旧项目中真正可验证的工程能力迁移到新项目主线。

---

## 一、项目定位映射

| 旧校级项目 | 新国省级项目 |
|---|---|
| 绿链未来：基于“两链一桥”的分布式光伏 RWA 资产数字化管理平台 | 绿链未来 / SolarMRV：基于国密与联盟链的分布式光伏 MRV 可信数据底座 |
| 光伏 RWA 普惠交易平台 | 分布式光伏减排量可信核算与核查数据服务平台 |
| 解决融资难、流动性差 | 解决 MRV 数据不可信、核查成本高、数据难复用 |
| 面向学生、小额投资者、Web3 绿色基金 | 面向 CCER 咨询机构、第三方核查机构、园区碳管理部门、出口企业 |
| 资产收益权拆分与流转 | 发电数据可信采集、减排量核算、报表输出、审计留痕 |

## 四、申请书章节映射

| 旧申请书章节 | 新申请书写法 |
|---|---|
| 项目简介 | 500 字以内，写 MRV 可信数据底座，不写 RWA 交易 |
| 项目概况 | 写双碳、CCER、MRV、分布式光伏数据可信痛点 |
| 管理模式 | 保留团队角色，改成技术、政策、试点、财务、视觉材料分工 |
| 生产或运营 | 写设备采集、签名、链上存证、MRV 报表、核查接口、PoC 试点 |
| 投融资方案 | 写大创经费用途、资源需求、后续服务收入，不写融资发行 |
| 效益预测 | 写数据服务收入、社会效益、教育实效，不写资产交易额 |
| 风险预测 | 写政策、技术、试点、数据安全、团队风险和合规红线 |
| 项目特色 | 写真实硬件、国密、联盟链、MRV、主动合规转向、国家战略契合 |
| 项目计划安排 | 改为 2026.05-2027.05 四阶段 |
| 预期成果 | 系统原型、MRV 报表、试点证明、软著、专利、调研报告 |
| 经费预算 | 硬件、云服务、调研差旅、知识产权、材料制作、测试服务 |

## 五、项目名称候选映射

### 推荐 1

`绿链未来：基于国密与联盟链的分布式光伏 MRV 可信数据底座`

优点：

- 保留旧项目“绿链未来”的连续性。
- 明确国密、联盟链、分布式光伏、MRV。
- 避开 RWA / 代币 / 交易。

### 推荐 2

`SolarMRV：面向分布式光伏的减排量可信核算与碳资产数据底座`

优点：

- 与最新 BP 一致。
- 碳核算和 MRV 主线更清楚。
- 更适合后续省级 / 国家级材料。

### 推荐 3

`绿链光伏：面向 CCER 核查的分布式光伏 MRV 数据服务平台`

优点：

- 直接贴近 CCER 核查。
- 比较像正式项目。

风险：

- 如果短期内没有真实核查机构 PoC，写“面向 CCER 核查”要注意不要夸大为已经采信。

## 六、新旧项目关系的一句话解释

可用于申请书或答辩：

> 本项目是在原校级“绿链未来”分布式光伏可信采集与链上存证原型基础上的升级版本。团队保留了已验证的物联网采集、国密签名、联盟链存证、自动监测和可视化能力，同时根据最新监管环境主动剥离旧版 RWA 金融化叙事，将项目正式转向分布式光伏减排量 MRV 可信数据底座，服务 CCER 申报、第三方核查、园区碳管理和出口企业绿电证明等合规场景。

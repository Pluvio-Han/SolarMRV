# 项目结构说明

本文件记录当前项目文件夹的整理方式。此次整理只做归类移动和说明补充，没有删除项目内容。

## 顶层目录

- `README.md`：项目主说明与实验报告内容。
- `api/`：后端 API 服务入口。
- `assets/`：README 和报告引用的关键展示图片。
- `charts/`：光伏监测生成的图表图片。
- `contracts/`：智能合约与 ABI 文件。
- `data/`：采集数据、钱包等运行数据。
- `frontend/`：前端工程。
- `logs/`：运行日志文件。
- `reports/`：零散 HTML 或报告导出文件。
- `scripts/`：人工执行的运维、设备控制和报告生成脚本。
- `tests/`：链上、设备、重启、上传等测试脚本。
- `utils/`：辅助工具和调试工具。
- `政策资料/`：政策、监管、绿证、数字金融等资料。
- `比赛材料/`：商业计划书、正文、图表素材、Word 定稿和数据搜集材料。
- `项目总结报告内容/`：项目总结、PDF、DOCX 和相关正式报告材料。

## 运行提示

核心 Python 模块仍保留在项目根目录，例如 `chain_client.py`、`solar_monitor.py`、`wallet_manager.py`、`config.py` 等，以避免破坏原有导入方式。

已经移动到 `tests/` 和 `scripts/` 的 Python 文件都加入了项目根目录路径兼容逻辑，可以继续从项目根目录直接运行，例如：

```bash
python tests/test_chain_client.py
python scripts/turn_on_light.py
bash scripts/start_nodes.sh
```

## 整理原则

- 不删除任何项目内容。
- 不移动已有工程目录内部结构，例如 `frontend/`、`contracts/`、`data/`、`比赛材料/`、`政策资料/`。
- 根目录保留核心运行模块，散落的测试、脚本、日志和导出物按用途归类。

# 朋友 BP 格式拆解参考说明

参考文件：
`/Users/evanhan/项目/光伏项目/项目总结报告内容/信评要素商业计划书(3).docx`

## 结论

这份文档适合被当作“规范结构参考”，不适合被当作“视觉成品模板”直接照搬。

它的优势在于：
- 结构完整
- 编号规范
- 长文档感强
- 正式度足够

它的不足在于：
- 传统感偏重
- 图文节奏偏慢
- 文字密度较高
- 不够突出技术产品项目的现代感

## 可直接借用的部分

### 1. 前置结构
这份文档有典型正式 BP 的前置内容：
- 封面
- 目录
- 图目录
- 表目录

这套结构应直接借到 SolarGuard 第二版中。

### 2. 标题层级
文档采用了典型的三级结构：
- 一级：`第一章`
- 二级：`1.1`
- 三级：`1.1.1`

这套层级是正式比赛型长文档最稳的做法。

### 3. 页码分离
从 docx 结构看：
- 存在 `w:pgNumType`
- 存在 header/footer 引用

说明其采用了前置页与正文分离的正式页码逻辑。SolarGuard 终稿应采用相同方法。

### 4. 字体基底
从样式 XML 可以确认：
- 中文默认字体包含 `宋体`
- 英文默认字体包含 `Times New Roman`

这说明朋友那份文档使用的是非常标准的正式文档字体基底。

## 不建议直接照搬的部分

### 1. 全文都走传统字体气质
如果 SolarGuard 也把标题全部做成传统宋体风格，文档会显得偏旧，不够符合“新能源可信数据平台”这种项目的视觉身份。

### 2. 过于学术化的文字密度
朋友那份更像研究型计划书。SolarGuard 当前已有较强图片体系，不应该退回到大段文字堆叠模式。

### 3. 节奏较慢
朋友那份的阅读体验偏“长报告”。SolarGuard 应保持正式，但要让首页、产品页、技术页更有图文节奏。

## 对 SolarGuard 的具体借用方案

### 结构层
直接借：
- 封面顺序
- 前置页构成
- 标题编号方法
- 页码分离逻辑
- 图表目录逻辑

### 视觉层
保留我们自己的：
- 绿色主视觉
- 已做好的技术结构图
- 更现代的标题字体
- 更强的项目感和产品感

### 文档气质
最终目标不是做成“融资 PPT”，也不是做成“纯论文”，而是做成：
`正式比赛计划书 + 现代项目表达`

## 推荐最终折中方案

- 封面和目录：参考朋友 BP
- 页码和编号：参考朋友 BP
- 正文字体：参考朋友 BP 的正式基底
- 标题字体和图文节奏：采用 SolarGuard 当前方案
- 图表数量和质量：优于朋友 BP

## 当前建议

对 SolarGuard 第二版，最合适的处理是：
1. 结构向朋友 BP 靠拢
2. 视觉不向朋友 BP 退回
3. 图片保留我们已经做好的成熟版本
4. 第七章高文字密度内容改 Word 原生页，不强做 PNG

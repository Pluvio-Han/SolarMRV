# 财务资源组任务清单

## 任务目标

补齐商业计划书中“项目现在真实投入了什么、未来三阶段需要多少资源、这些资源分别用来做什么、收入逻辑为什么合理”这几个部分的真实支撑材料。

本组不是做复杂财务模型，而是把项目现阶段和后续推进阶段的真实成本与资源需求整理清楚。

## 基本原则

1. 不夸大，不虚构
2. 能用真实票据、购买记录、云平台账单支撑的，优先用真实记录
3. 不能精确还原的，用 `保守估算`，并注明估算依据
4. 所有数字要能解释“为什么需要这笔投入”

## 一、已投入资源搜集

### 需要搜集的内容

1. 硬件设备投入
   - 光伏控制器
   - 通信线材
   - 可能使用过的采集设备、模块、转接设备
2. 服务器与运行环境投入
   - 云服务器
   - 域名
   - API、测试环境、存储空间
3. 开发与展示投入
   - 打印、装订、海报
   - PPT、设计、图表工具
   - 比赛报名与路演物料
4. 时间投入
   - 每位成员的大致开发/写作/调研时间

### 建议搜集方式

- 电商订单截图
- 支付记录
- 云平台账单截图
- 成员自报时间投入表

### 最终交付

- `已投入成本汇总.xlsx`
- `已投入凭证截图` 文件夹
- `成员时间投入表.xlsx`

## 二、后续三阶段预算设计

### 阶段 1 比赛与立项推进阶段

#### 需要细化的支出项

1. 商业计划书与 PPT 优化
2. 图表、架构图、页面展示图制作
3. 打印与答辩物料
4. 材料整理与传播成本

#### 需要给出的数据

- 每项预计金额区间
- 为什么需要这项费用
- 是否可以用现有资源覆盖

### 阶段 2 试点验证阶段

#### 需要细化的支出项

1. 硬件重新接入和运输
2. 服务器与运行维护
3. 系统优化与接口联调
4. 小范围试点场景支持
5. 数据持续采集与验证成本

#### 需要给出的数据

- 每项预计金额区间
- 是否一次性投入或持续性投入
- 哪些投入是试点必须项

### 阶段 3 平台化升级阶段

#### 需要细化的支出项

1. 多设备兼容与接口扩展
2. 前后端产品化完善
3. 可视化与报告模块标准化
4. 对接 ESG / 绿证 / 绿色金融场景的专项开发
5. 更系统的对外展示与合作成本

#### 需要给出的数据

- 每项预计金额区间
- 为什么这阶段投入明显更高
- 哪些费用和未来复制性直接相关

### 最终交付

- `未来三阶段预算表.xlsx`
- `预算依据说明.docx`

## 三、收入逻辑支撑材料

### 需要搜集的内容

1. 同类技术服务、SaaS、数据服务、系统部署服务的收费参考
2. 项目未来可能的三类收入路径的可解释依据
   - 基础服务收入
   - 报告/核验收入
   - 定制合作收入
3. 每类收入更适合在什么阶段出现

### 建议搜集关键词

- `能源管理 SaaS 价格`
- `数据服务 平台 收费`
- `定制开发 报价 参考`
- `ESG 数据服务 收费`
- `运维平台 服务费`

### 最终交付

- `收入路径参考表.xlsx`
- `收费案例摘录.docx`

## 四、资源结构支撑材料

### 需要搜集的内容

1. 项目推进最关键的四类资源分别是什么
   - 场景资源
   - 技术资源
   - 专家资源
   - 展示传播资源
2. 为什么这些资源比单纯现金更关键
3. 哪类资源目前最缺
4. 哪类资源最先补齐能带来最大推进效果

### 最终交付

- `资源需求说明.docx`
- `资源优先级排序表.xlsx`

## 五、建议的表格字段

### 已投入成本表字段

- 支出类别
- 具体项目
- 金额
- 时间
- 支付方
- 是否有凭证
- 用途说明

### 预算表字段

- 阶段
- 预算项目
- 预计金额
- 预算依据
- 是否必须
- 备注

### 收入逻辑表字段

- 收入类型
- 对应产品/服务
- 适合阶段
- 参考依据
- 说明

## 六、优先级

### 第一优先级

1. 已投入成本汇总
2. 三阶段预算表
3. 资源优先级排序

### 第二优先级

1. 收费参考案例
2. 成员时间投入统计
3. 未来收入路径说明

- `已投入成本`

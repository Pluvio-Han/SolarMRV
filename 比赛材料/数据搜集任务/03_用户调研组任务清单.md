# 用户调研组任务清单

## 任务目标

补齐商业计划书中“用户是谁、用户痛点是什么、用户是否真的需要这类系统、项目的使用场景是否成立”这四个问题的真实依据。

本组最重要的任务不是做漂亮问卷，而是拿到能够支持 BP 的真实需求证据。

## 调研对象优先顺序

1. 分布式光伏运维或项目管理相关人员
2. 学校、园区、企业能源管理相关老师或工作人员
3. ESG、审计、项目申报、绿色金融相关老师或从业者
4. 熟悉新能源项目申报、绿证、节能管理的人

## 一、访谈对象类型与目标

### 1. 光伏运维或设备管理侧

#### 想验证的问题

1. 他们现在怎么获取设备数据
2. 现有监控方式有什么痛点
3. 数据有没有丢失、断档、整理麻烦的问题
4. 他们是否需要“可信留痕”和长期记录
5. 对自动监测、自动出图、自动报告是否有兴趣

#### 建议问题

- 你们现在如何查看光伏设备运行数据
- 平时最常关注哪些指标
- 现在的数据记录方式是否连续、是否容易导出
- 如果要做汇报或审计，数据整理麻烦吗
- 你认为“可信留痕”和历史追溯有必要吗
- 如果有一个系统能自动采集、自动保存、自动出图，你觉得是否有用

### 2. 园区 / 企业 / 学校能源管理侧

#### 想验证的问题

1. 是否需要汇总多个设备或项目的数据
2. 是否需要标准化能源数据
3. 是否关心节能降碳、绿色评价、项目汇报
4. 是否需要更统一的数据治理能力

#### 建议问题

- 你们会不会定期整理能源相关数据
- 这些数据是分散的还是统一的
- 在做节能、双碳、能源管理汇报时，最麻烦的是什么
- 如果底层设备数据更可信、格式统一，对你们有帮助吗
- 你们会不会需要历史留痕和自动化图表

### 3. ESG / 审计 / 项目申报 / 绿色金融侧

#### 想验证的问题

1. 他们是否需要可信底层数据作为支撑
2. 现在核验新能源项目数据难在哪里
3. 如果有可追溯的设备层数据，是否能降低沟通成本
4. 对“联盟链存证”“可信数据来源”是否认可

#### 建议问题

- 在 ESG、尽调、审计或项目申报中，新能源数据是否重要
- 你们通常怎么判断数据真实性
- 最难核验的是哪些部分
- 如果有设备采集、签名、存证、历史记录一整套链路，你会觉得更有说服力吗
- 你认为这样的系统最适合服务哪些正式场景

## 二、最低样本要求

### 最低目标

- `5 人深度访谈`

### 理想目标

- `8 到 12 人`，覆盖至少 `3 类对象`

### 每份访谈至少记录

1. 访谈对象身份类型
2. 场景背景
3. 现有工作方式
4. 当前痛点
5. 对项目功能的反馈
6. 是否愿意试用或认为谁会需要

## 三、可选问卷调研

如果时间允许，可以做一份简短问卷。

### 问卷目标

用于做基础统计，不代替深度访谈。

### 可设计题目

1. 你是否接触过光伏、能源管理、ESG 或绿色项目数据
2. 你认为这些数据最常见的问题是什么
3. 你是否认为“自动采集 + 长期记录 + 可追溯”有价值
4. 你最看重下列哪项能力
   - 自动监测
   - 历史记录
   - 图表生成
   - 可信存证
   - 报告输出
5. 你认为最适合使用本项目的主体是谁
   - 运维方
   - 园区/企业
   - 学校/科研场景
   - 审计/ESG 机构
   - 金融相关机构

## 四、交付格式

### 1. 访谈纪要表

每位访谈对象一行，字段包括：
- 编号
- 对象类型
- 场景背景
- 当前痛点
- 最关注能力
- 对项目价值判断
- 可引用原话

### 2. 高频痛点汇总表

统计最常出现的痛点，例如：
- 数据不连续
- 整理麻烦
- 缺少可信留痕
- 汇报和审计难
- 缺少统一展示和输出

### 3. 典型用户画像总结

至少整理 `3 类用户画像`：
- 运维与项目管理用户
- 园区/企业能源管理用户
- ESG / 审计 / 金融辅助用户

### 4. 一页调研结论摘要

建议直接输出这几类结论：
- 谁最需要
- 他们现在怎么做
- 他们最痛的点是什么
- SolarGuard 的哪些模块最有价值
- 哪些场景最适合作为初期试点

## 五、如何让数据可直接进入 BP

每一条结论尽量整理成这种表达：

- `访谈对象类型`：某园区能源管理老师
- `问题`：日常能源数据分散、汇报整理耗时
- `反馈`：如果底层设备数据能自动采集和留痕，汇报可信度会更高
- `可用于章节`：第三章用户对象 / 第五章应用场景 / 第六章推广路径

## 六、优先级

### 第一优先级

1. 深度访谈 5 到 8 人
2. 高频痛点总结
3. 三类用户画像

### 第二优先级

1. 问卷统计
2. 场景优先级排序
3. 试点合作意向判断

## 七、建议文件夹结构

- `访谈纪要`
- `问卷结果`
- `用户画像`
- `结论摘要`

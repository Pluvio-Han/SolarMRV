# 第一章 项目概述

## 1.1 项目背景

在国家“双碳”战略、新型电力系统建设和数字中国建设持续推进的背景下，分布式光伏正在成为我国新能源体系中增长最快、覆盖最广的基础场景之一。与集中式能源项目相比，分布式光伏具有部署灵活、贴近负荷侧、应用范围广等优势，正在园区、校园、工商业屋顶和居民场景中加速落地。然而，伴随分布式光伏规模快速扩张，底层运行数据的可信获取、持续记录、跨主体核验和场景化使用问题也日益突出。

当前，大量分布式光伏项目虽然已经具备基础监测能力，但真实发电数据仍普遍存在采集链路分散、数据格式不统一、连续性不足、缺乏可信留痕和难以被第三方机构采信等问题。这些问题不仅影响设备运维效率，也限制了发电数据在绿证衔接、ESG 披露、绿色金融辅助决策、项目审计和风险管理等场景中的进一步应用。换句话说，新能源产业中并不缺“数据”，真正稀缺的是“可信、连续、可追溯、可验证、可复用的数据”。

与此同时，国家正持续推进数据要素市场化配置改革、可信数据空间建设、数字金融服务实体经济和绿色金融体系完善。分布式光伏发电数据作为新能源产业中的重要基础数据，正在从单纯的运行信息逐步转变为具有治理价值、金融价值和合规价值的关键数据资源。如何将底层设备数据转化为可信业务数据，并进一步服务于绿色资产核验和绿色金融场景，已经成为一个具有现实意义的创新方向。

基于上述背景，`SolarGuard` 项目围绕分布式光伏场景，尝试构建一套从物理设备采集到数据签名、联盟链存证、自动监测告警、可视化展示，再到绿色资产核验服务的完整原型系统。项目希望通过技术手段解决“数据可信性不足”这一底层问题，为新能源数据要素化应用提供可验证、可扩展的技术路径。

## 1.2 项目要解决的核心问题

`SolarGuard` 聚焦的并不是光伏硬件制造本身，也不是单纯的区块链概念展示，而是分布式光伏数据在真实应用过程中面临的核心障碍。

首先，分布式光伏项目底层数据来源复杂，设备型号、通信接口、部署环境和运维方式差异较大，导致数据采集结果难以标准化整合。许多数据虽然被设备本地记录，却没有形成统一、结构化、可长期使用的数据资产基础。

其次，光伏运行数据缺乏可信确权机制。传统监控平台大多只解决“看得到”的问题，却难以解决“能证明、可追溯、可核验”的问题。一旦数据需要被金融机构、审计主体、合作方或监管场景引用，就容易面临真实性不足、留痕不完整、来源不可追溯等问题。

再次，分布式光伏数据与绿证、ESG、绿色金融之间仍然存在明显的场景断层。当前许多新能源项目的数据停留在设备运维层，没有真正进入跨主体协同使用的业务层。金融机构难以低成本获取可信底层数据，ESG 披露和绿色审计缺乏自动化、连续化的支撑工具，数据的价值难以持续释放。

最后，在政策环境变化背景下，新能源资产数字化项目还面临合规叙事的挑战。如何在不触碰金融监管高风险边界的前提下，将技术创新与真实应用场景结合起来，是当前项目设计和项目申报必须同时回应的问题。

因此，`SolarGuard` 要解决的根本问题可以概括为：如何为分布式光伏场景建立一套可信、连续、可验证、可服务多主体的数据基础设施，使设备侧的原始发电数据能够真正转化为可被应用场景采信和复用的可信业务数据。

## 1.3 项目目标

围绕上述问题，`SolarGuard` 项目设置了三个层次的目标。

第一，构建面向分布式光伏场景的可信数据采集与处理链路。项目通过接入光伏控制器等设备，采集发电功率、电压、电池状态、累计发电量等关键指标，并进行结构化处理，为后续确权、存证和应用提供统一的数据基础。

第二，构建面向多方协同使用的可信存证与追溯机制。项目引入数字签名和联盟链存证技术，使关键数据在产生后能够形成可验证、可追溯、可长期保存的记录，为后续核验、审计和业务调用提供依据。

第三，探索新能源数据服务于绿色资产核验和绿色金融辅助场景的应用路径。项目不将重点放在公开金融化运作，而是放在更符合当前政策方向的场景上，例如绿证衔接、ESG 披露、绿色贷款和保险辅助尽调、园区能源数据治理等，从而让底层数据真正进入更高价值的业务链条。

从技术原型到比赛申报的当前阶段，项目目标并不是直接建成大规模商业平台，而是完成一套具有真实实验基础、完整系统链路和明确扩展方向的原型系统，并在此基础上形成可持续深化的项目方案。

## 1.4 项目亮点

与单纯的软件平台或概念化方案相比，`SolarGuard` 的亮点主要体现在“真实场景、可信链路、系统闭环、政策适配”四个方面。

第一，项目建立在真实分布式光伏设备采集基础之上，已经围绕实际硬件控制器完成了数据读取、状态监测和连续记录相关实验，具有明确的物理世界锚点，而非停留在纯概念推演层面。

第二，项目强调可信数据机制而不是普通数据展示。通过数字签名、联盟链存证、自动监测和多模块联动，项目将“看板展示型监控”提升为“具备可验证能力的数据底座”，增强了数据在跨主体场景中的可信度和复用潜力。

第三，项目已初步形成从硬件采集、边缘处理、链上记录到前端可视化的完整系统原型。现有代码和实验结果表明，项目并非单点模块开发，而是已经完成了关键链路的闭环验证。

第四，项目主动根据当前政策环境调整技术叙事和应用定位，将重点放在新能源可信数据、绿色资产核验和绿色金融辅助服务上，既保留了技术创新性，又增强了比赛申报和后续落地的合规适配性。

## 1.5 项目当前进展与已取得成果

目前，`SolarGuard` 项目已经完成了第一阶段的原型系统搭建和核心能力验证，具备形成正式商业计划书和参赛材料的基础。

在硬件侧，项目已完成针对光伏控制器数据的采集实验，能够获取发电功率、光伏电压、电池电量、累计发电量等关键指标，并支持模拟模式下的数据生成，为后续开发和展示提供了双轨验证条件。

在数据处理与可信机制方面，项目已实现关键数据的签名处理和结构化封装，并构建了与联盟链交互的基础能力，支持将采集结果进行链上记录和查询验证，形成可追溯的数据记录链。

在系统功能方面，项目已开发自动监测程序，能够完成周期采样、数据落盘、图表生成、消息推送和链上同步等任务，初步具备持续监测和自动运行能力。同时，项目还搭建了 API 与前端原型，能够对关键数据进行可视化展示，呈现系统整体运行逻辑。

在材料成果方面，项目已形成代码仓库、实验脚本、项目总结报告、原型展示材料、政策资料库等多种成果形式，为后续商业计划书撰写、路演 PPT 制作和比赛申报提供了系统性支撑。

总体而言，`SolarGuard` 当前已经从“想法提出阶段”进入“原型验证与材料整合阶段”。项目虽然仍有若干接口和业务场景需要继续完善，但核心技术链路已经明确，项目定位已经逐步稳定，具备进一步打磨为高质量创新创业参赛项目的条件。

## 1.6 项目定位

基于当前技术完成度、政策环境和比赛申报要求，`SolarGuard` 的正式项目定位为：

`面向分布式光伏场景的可信数据采集、联盟链存证与绿色资产服务平台`

项目的核心不是做光伏硬件本体，也不是面向公众开展代币交易，而是通过物联网、密码学和联盟链等技术，为新能源数据建立可信采集、可信留痕和可信核验能力，进一步服务于绿证衔接、ESG 披露、绿色金融辅助决策和项目审计等场景。

这一定位既准确反映了项目现有成果，也为后续商业化探索和比赛评审提供了更加稳健、清晰和可持续的叙事基础。
